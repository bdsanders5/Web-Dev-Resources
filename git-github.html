Using Git & Github

Git is a version control system that allows you to capture snapshots of a project.

Setting up & pushing to repositories:

Assumptions: You have git installed... If not, Download & Install from git-scm

  • Initialize local repository by opening terminal, cd to project directory & run $git init
  • Add name to config with $git config --global user.name 'Your Name'
  • Add email to config with $git config --global user.email 'YourEmail@you.com'
  • Add file to staging area (index) with $git add
    or all files with $git add .
  • check staging area items and status anytime with $git status
  • Commit changes in staging area with $git commit -m 'comment about changes' (pushes to local repository)
  • Push local repository to remote repository (github) with $git push

Pulling from remote repository

  • cd to project directory
  • run $git pull for existing projects latest version
  • for downloading new remote project into directory, run $git clone

Excluding files / directories

  • Create a file named .gitignore in your working directory
  • Add file names & / or directory names preceded with f slash one per line ie...
      
                filename1.js
                /dirName1
                filename2.js
                ...etc...

Branching

Branching enables changes without affecting master code base. Great for adding new features / function to codebase. Especially in team environments.

Example of Branch for login feature might go as follows:

  • Add new branch with $git branch login
  • Switch branche to login with $git checkout login
  • You can see which branch you're in with $git status
  • Stage & commit as usual. Changes / additions will only affect current branch
  • Can switch back & forth between branches with $git checkout
  • If changes shoud be added to master, switch back with $git checkout master & then merge with $git merge login (vim will open / ... press i, add comment, press esc, type :wq
